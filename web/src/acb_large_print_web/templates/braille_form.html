About Braille Studio

Translate English text to braille, or back-translate braille to English text. Translation follows BANA (Braille Authority of North America) standards via liblouis {{ louis_version }}.

  • UEB (Unified English Braille) is the current BANA literary standard, adopted January 4, 2016.
  • BRF output is wrapped at the BANA-standard 40 cells per line, ready for embossers.
  • Back-translation is approximate for contracted braille (Grade 2) because contractions are lossy by design.
  • EBAE (legacy) tables are provided for interoperability with pre-2016 materials only.

BANA Standards Reference

Unified English Braille (UEB)
The current BANA literary braille standard, adopted January 4, 2016. Replaces the English Braille American Edition (EBAE). Covers literary text, technical notation, and foreign language passages.
BANA Formats
BANA formatting guidelines specify 40 cells per line and 25 lines per page for standard braille paper (11″ × 11.5″ interpoint or 11″ × 11″ single-sided). BRF output from this tool uses 40-cell line wrapping per these guidelines.
Computer Braille Code (CBC)
BANA-approved 8-dot code for computer notation, source code, file paths, and technical content. Provides a one-to-one ASCII mapping without ambiguity.
Nemeth Code
BANA-approved braille mathematics code. Nemeth requires structured input with explicit math-mode delimiters and is beyond the scope of plain-text translation. For Nemeth transcription, use the liblouis Nemeth tables via a dedicated math document processor such as MathType or LAMBDA.
