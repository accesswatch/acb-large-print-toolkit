What does the audit do?

The audit checks your document against the ACB Large Print Guidelines and WCAG 2.2 AA requirements. Rules adapt automatically to the file type:

  • Word (.docx) -- fonts, sizes, spacing, alignment, emphasis, headings, margins, and document properties.
  • Excel (.xlsx) -- sheet names, table headers, merged cells, alt text, hidden content, and color-only cells.
  • PowerPoint (.pptx) -- slide titles, reading order, alt text, font sizes, speaker notes, and chart descriptions.
  • Markdown (.md) -- heading hierarchy, emphasis violations, link text, alt text, emoji, em-dashes, and tables.
  • PDF (.pdf) -- title, language, tagging, font sizes, font families, scanned pages, and bookmarks.

You receive a detailed report showing each finding with its severity and the relevant guideline reference.

Understanding severity levels
Critical
Issues that make the document unreadable for low-vision users or completely block assistive technology. Must be fixed. Examples: wrong font, text smaller than 18pt, missing slide titles, images without alt text.
High
Significant readability or structure issues. Should be fixed. Examples: wrong heading sizes, justified alignment, skipped heading levels, blank column headers.
Medium
Compliance gaps that affect quality but may not block readability. Examples: incorrect line spacing, merged cells, reading order mismatches, generic sheet names.
Low
Minor issues or best-practice recommendations. Examples: widow/orphan control, header row not frozen, missing speaker notes.
